PARTIE B - ANALYSE STATIQUE (PHPStan)
====================================================================

1) Installer/verifier PHPStan
Commande:
  composer require --dev phpstan/phpstan
  vendor\bin\phpstan --version

2) Configuration
Fichier:
  phpstan.neon
Contenu minimum:
  includes:
      - phpstan-baseline.neon
  parameters:
      level: 5
      paths:
          - src

Explication:
- level 5: niveau intermediaire.
- paths: dossiers analyses.
- baseline: neutralise temporairement les erreurs deja connues.

3) Premiere analyse
Commandes:
  vendor\bin\phpstan analyse src
  vendor\bin\phpstan analyse src\Controller
  vendor\bin\phpstan analyse src\Service

4) Analyse ciblee (deja executee)
Commande:
  php .\vendor\bin\phpstan analyse -c phpstan.neon src\Service\QuizResultService.php src\Service\RecommandationService.php tests\Service\QuizResultServiceTest.php tests\Service\RecommandationServiceTest.php
Resultat attendu:
  [OK] No errors

5) Monter le niveau (atelier)
- Modifier:
  level: 8
- Relancer:
  vendor\bin\phpstan analyse

6) Ignorer temporairement certaines erreurs (si demande atelier)
Dans phpstan.neon:
  parameters:
      ignoreErrors:
          - '#Call to an undefined method#'
Note:
  Utiliser temporairement, puis supprimer progressivement.

7) Plan de correction reel (recommande)
- Corriger par lots:
  Lot 1: erreurs de types DateTime (setDate/setTime int)
  Lot 2: getUser() non type -> helper utilisateur type
  Lot 3: injections de dependances manquantes
  Lot 4: signatures de methodes/retours
  Lot 5: logique/conditions toujours vraies

PARTIE C - DOCTRINE DOCTOR
====================================================================

1) Installation
Commande:
  composer require --dev ahmed-bhs/doctrine-doctor

Si conflit de dependances:
  composer require ahmed-bhs/doctrine-doctor:^1.0 webmozart/assert:^1.11 --with-all-dependencies
  composer require --dev ahmed-bhs/doctrine-doctor

2) Config bundle (si non detecte)
Fichier:
  config/bundles.php
Ligne attendue:
  AhmedBhs\DoctrineDoctor\DoctrineDoctorBundle::class => ['dev' => true, 'test' => true],

3) Correction importante deja faite
Fichier:
  config/packages/doctrine_doctor.yaml
Point cle:
  doctrine_doctor:
      enabled: true
Explication:
  Evite l'erreur "Expected bool, got string".

4) Utilisation
- Demarrer l'app en dev
- Ouvrir une page
- Ouvrir Symfony Web Profiler
- Ouvrir panel Doctrine Doctor
- Capturer:
  integrity, security, configuration, slowest queries

5) Boucle de correction
Apres chaque correction:
  php bin/console cache:clear
  relancer serveur dev
  reverifier panel Doctrine Doctor
